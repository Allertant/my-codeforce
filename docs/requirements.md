# my-codeforces 本地助手需求规格说明书 (v2.0)

## 1. 核心目标
构建一个专为 Codeforces 刷题设计的“本地 IDE” Web 界面。它能自动完成从**查题、分级、抓取样例、编写代码、本地评测、到清洗提交**的全生命周期管理，彻底解决频繁切换窗口、手动复制模板、以及多解法管理混乱的痛点。

## 2. 核心功能与逻辑

### 2.1 题号输入与自动化工作流
1. **智能启动：** 用户在 Dashboard 或编辑器顶部输入题号（如 `78A`）。
2. **并行加载（Suspense 风格）：** 
   - **编辑器：** 立即加载 `template.py` 供用户开始编写。
   - **后端：** 启动异步抓取任务。
3. **信息自动填充：** 
   - 联网查询题目的 Rating（难度）和 Tags（标签），设置合理超时时间。
   - 成功则按 Rating 自动定位目录（如 `800/`），失败则允许用户手动选择。
   - 抓取 Sample Input/Output，实时注入页面测试区域。
4. **存量兼容：** 打开不符合新规范的旧代码时，弹出提示引导用户一键升级（注入 Metadata 标签，选项 C）。

### 2.2 数据存储规范 (Source of Truth)
1. **代码文件：** `<难度目录>/<题号>_<版本>.py`（例如 `800/78A_v1.py`）。
2. **测试数据（共享模式）：** 同一题号共享一个 `<难度目录>/<题号>.yaml` 文件（选项 A）。
   - YAML 格式采用 `|` 语法处理多行字符串，确保输入输出肉眼可见、易于手动修改。
3. **Metadata（代码内嵌）：** 通过 `# === METADATA ===` 标签划定区域，存储题目名称、Rating、时间及 `Status: AC` 等状态（选项 B）。

### 2.3 在线编辑与版本演进
1. **编辑器：** 集成 Monaco Editor，支持 Python 语法高亮与补全。
2. **版本管理（LeetCode 风格）：** 
   - 编辑器内提供版本切换列表，支持在 `v1`, `v2` 间快速跳转。
   - **自动暂存：** 切换版本或离开页面时，系统自动保存当前未保存的内容。
   - **开启新版本：** 提供“从当前代码克隆”或“使用纯净模板”两个入口（选项 C）。

### 2.4 本地运行与评测
1. **执行环境：** 智能感知 `.venv` 虚拟环境（选项 C）；工作目录隔离在题目所在文件夹。
2. **评测引擎：** 
   - **混合模式：** 一键跑完所有 YAML 样例，展示 AC/WA 汇总列表。
   - **详细对比：** 点击失败用例显示 Diff 视图，对比预期输出与实际输出。
   - **极简对比：** 仅进行文本层面的极简对比，忽略行末空格与空行。
   - **超时控制：** 默认设置 2s 超时保护，防止死循环导致后端阻塞。
3. **macOS 容灾运行：** 提供“复制命令行”按钮，生成 `printf 'code' > /tmp/cf_run.py && python3 /tmp/cf_run.py` 格式的命令，方便在终端脱离 Web 界面运行（选项 1）。

### 2.5 标记与提交
1. **代码清洗：** 点击“复制提交”按钮时，系统根据标签自动剔除 `# === LOCAL_DEBUG ===` 区域，仅保留清洗后的代码（保留姓名、时间等元数据）。
2. **快速提交：** 提供一键跳转到 Codeforces 该题 Submit 页面的按钮。
3. **标记 AC：** 支持手动在代码中修改 `Status` 标签，也支持在 Web 界面点击按钮一键回写 `Status: AC`。

## 3. 界面交互 (UX)
1. **Dashboard (概览视图)：** 
   - 展示最近刷题记录、按难度分级的统计小部件。
   - 支持按题号搜索、按修改时间或难度排序。
   - **Resume 模式：** 再次打开 Web UI 时，自动恢复到上次未完成的题目和版本（选项 1）。
2. **错误处理：**
   - YAML 格式损坏时拒绝测试并报错。
   - 抓取失败时提供“手动录入”和“一键重试”入口。
   - 限制执行输出缓冲区大小，防止内存溢出。

## 4. 技术约束
1. **后端：** 推荐使用 FastAPI (Python)，负责文件系统操作、抓取逻辑及子进程管理。
2. **前端：** 推荐使用 React + Monaco Editor，利用 Suspense 处理异步抓取状态。
3. **安全性：** 后端执行命令需限制工作目录在本仓库下。

## 5. 接口设计草案
- `POST /api/init`: 输入题号，启动抓取并返回文件路径与初始内容。
- `POST /api/save`: 暂存/保存当前编辑器内容。
- `POST /api/run`: 调用本地 Python 运行测试。
- `GET /api/history`: 获取 Dashboard 所需的题目列表及统计数据。
- `POST /api/mark_ac`: 更新代码文件中的状态为 AC。
