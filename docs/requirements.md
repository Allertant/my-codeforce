# my-codeforces 需求文档 (v2.1)

## 1. 业务需求总结
当前刷题流程存在频繁切换编辑器、手动复制模板、缺少 Python 语法提示、以及测试数据无法持久化等痛点。本系统旨在提供一个本地 Web UI，让用户在浏览器中一站式完成“输入题号-获取题目-编写代码-本地自测-清洗提交”的闭环。

## 2. 目标与范围
### 2.1 目标
- **自动化起步：** 输入题号自动分类并抓取样例。
- **一致性体验：** 在 Web 界面提供具有语法提示的编辑器。
- **本地评测：** 一键运行本地 Python 解释器并比对输出。
- **版本化管理：** 同一道题支持多种解法的快速切换与对比。
- **现场恢复：** 自动暂存代码，支持“继续上次练习”。

### 2.2 非目标
- 不接入 Codeforces 账号提交 API。
- 不做云端多用户或复杂沙箱系统。

## 3. 用户流程
1. **启动与进入：** 打开主页（Dashboard），展示最近刷题状态。
2. **新题挑战：** 
   - 输入题号（如 `78A`）。
   - 系统自动抓取题目信息并生成对应难度的目录。
   - 编辑器自动加载 `template.py`。
3. **循环迭代：** 
   - 编写算法。
   - 点击“运行测试”，查看所有 Sample 的 AC/WA 状态。
   - 如有需要，创建新版本（`v2`, `v3`）进行优化。
4. **收工提交：**
   - 点击“复制提交代码”，获得清洗后的算法代码。
   - 点击“跳转提交”，在官网完成提交。
   - 回到 Web UI，标记该版本为 `AC`。

## 4. 功能需求
- **题号感知：** 联网获取难度（Rating）并按现有目录结构归档。
- **多样例管理：** 支持自动抓取和手动录入测试样例，且支持持久化。
- **编辑器增强：** 支持 Python 语法高亮、自动补全、自动暂存。
- **评测反馈：** 类似 LeetCode 的评测列表，支持详细的输出对比。
- **历史记录：** 自动扫描本地文件，在 Dashboard 展示所有做过的题目及状态。

## 5. 验收标准
1. 输入题号后能快速（或异步）看到题目信息和样例。
2. 本地运行结果能实时反馈到页面。
3. 保存的代码文件符合仓库现有的层级规范。
4. 清洗出的代码可以直接在官网提交通过。
